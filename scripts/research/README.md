# scripts/research/

Auto-research routine for IEO-launch-audit. **Phase C of ADR 0002**
(v1.6.0+).

## Files

- **[PROTOCOL.md](PROTOCOL.md)** — execution guide for a Claude Code
  session invoked monthly via `/schedule`. Discovery wave → steelman
  wave → verification wave → consolidation → PR. ~60-100min wall-clock,
  ~$50-90 in Claude credits per pass.
- **[auto-pass.sh](auto-pass.sh)** — git + gh ship script. `init`
  subcommand sets up the research branch; `ship` subcommand commits +
  pushes + opens the PR.
- **`YYYY-MM-discovery-slate.md`**, **`YYYY-MM-steelman.md`**,
  **`YYYY-MM-verification.md`** — wave outputs (auto-generated by the
  Claude Code session following PROTOCOL.md; not committed until
  `ship`).

## How to invoke

### Manual (one-off)

```bash
# In a Claude Code session, cd to the skill repo, then:
bash scripts/research/auto-pass.sh init      # sets up branch
# Follow PROTOCOL.md steps 2-6 (dispatch subagents, consolidate).
bash scripts/research/auto-pass.sh ship YYYY-MM
```

### Scheduled (monthly cadence — recommended)

Configure via Claude Code's `/schedule`:

```
/schedule monthly-on-1st 09:00 "Run the IEO-launch-audit auto-research pass per scripts/research/PROTOCOL.md against github.com/thomasthinks/IEO-launch-audit"
```

The scheduled session reads `PROTOCOL.md`, executes waves 2-6, calls
`auto-pass.sh ship`. Maintainer reviews the resulting PR.

### Calibration

After 3 monthly passes, review per PROTOCOL.md § 10:
- 0 novel candidates 3x consecutive → drop to quarterly cadence.
- Pass cost >$150 avg 3x consecutive → tighten subagent budgets.
- >30% PROMOTE-tier candidates later prove wrong → tighten verification.

These are the testable triggers ADR 0002 declared.

## What this routine does NOT do

Per ADR 0001 + ADR 0002 invariants:

- **Never merges PRs.** Maintainer reviews + decides every candidate.
- **Never mutates skill checks directly.** Only writes research
  artifacts + opens a PR.
- **Never mutates consumer state files.** Per-repo `.ieo-audit-state.yml`
  is consumer-side; out of auto-research scope.
- **Never updates maintainer memory automatically.** Memory updates are
  PR-suggested via the PR body, not auto-applied.
